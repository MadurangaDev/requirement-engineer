# Definition of Done — {Project/Team Name}

*Note: DoD is an Agile/Scrum team-working-agreement, not an IEEE-standardized artifact — see `references/ieee-standards.md`. It's a checklist, not a specification.*

A story/requirement is Done only when every applicable item below is true. Don't let "done" silently mean "code written."

## Code
- [ ] Code complete and merged
- [ ] Peer reviewed
- [ ] Follows agreed coding standards

## Testing
- [ ] Unit tests written and passing
- [ ] Acceptance criteria (see `acceptance-criteria.md`) all verified, including edge/failure scenarios
- [ ] No known regressions introduced

## Requirements Traceability
- [ ] Traceability matrix updated (`traceability-matrix.md`)
- [ ] Requirement status moved to Approved/Baselined if applicable (see reasoning-framework.md maturity ladder)

## Documentation
- [ ] User-facing documentation updated if behavior changed
- [ ] Glossary updated if new terms introduced

## Non-Functional
- [ ] Performance targets met (if specified in NFRs)
- [ ] Security review done (if applicable)
- [ ] Accessibility checked (if applicable)

## Sign-off
- [ ] Product owner / stakeholder has accepted the result against the original story
